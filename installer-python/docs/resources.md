# 资源文件说明

## 目录概述

本文档详细说明 `installer-python` 目录下资源文件的功能和作用。

## 资源目录说明

### 🎨 icon/
**图标资源目录**

**功能描述：**
- 存储安装器程序的图标文件
- 支持多尺寸显示，适配不同分辨率
- 提供程序的可视化标识

**文件说明：**

#### multi_size.ico
**多尺寸图标文件**

**技术特性：**
- 包含多个尺寸的图标（16x16, 32x32, 48x48, 64x64, 128x128, 256x256）
- 支持高 DPI 显示
- 优化图标质量和文件大小

**使用场景：**
- 程序窗口标题栏图标
- 任务栏和开始菜单显示
- 文件资源管理器中的程序图标

**图标设计规范：**
- **风格**：简洁现代，符合浏览器扩展主题
- **颜色**：使用品牌色彩，保持视觉一致性
- **辨识度**：易于识别，与功能相关

### 🖼️ setp/
**安装步骤图片目录**

**功能描述：**
- 存储安装向导界面的步骤示意图
- 提供视觉引导，增强用户体验
- 展示安装流程和预期结果

**文件说明：**

#### 1.png - 5.png
**安装步骤示意图**

**图片内容：**
- **1.png**：欢迎界面和许可协议步骤
- **2.png**：安装路径选择步骤
- **3.png**：浏览器检测和配置步骤
- **4.png**：文件复制和安装步骤
- **5.png**：安装完成确认步骤

**设计规范：**
- **尺寸**：统一尺寸，适配界面布局
- **风格**：简洁明了，突出关键信息
- **一致性**：保持视觉风格和色彩统一
- **可读性**：文字清晰，图标易于理解

**使用方式：**
```python
# 在安装器中加载步骤图片
def load_step_image(self, step_number):
    image_path = f"setp/{step_number}.png"
    image = Image.open(image_path)
    photo = ImageTk.PhotoImage(image)
    return photo
```

### 📁 output/
**输出目录**

**功能描述：**
- 存储打包生成的可执行文件
- 作为发布版本的存储位置
- 提供版本管理和发布准备

**文件说明：**

#### ec-chrome-extension-installer.exe
**最终安装器程序**

**程序特性：**
- **独立性**：包含所有依赖，无需安装 Python
- **兼容性**：支持 Windows 7/10/11 系统
- **安全性**：数字签名验证（如适用）
- **性能**：优化启动速度和内存使用

**发布流程：**
1. 运行 `python build.py` 进行打包
2. 生成的 exe 文件自动保存到 output 目录
3. 进行功能测试和兼容性验证
4. 准备发布版本和更新说明

**版本管理：**
- 使用文件名包含版本号进行区分
- 保留历史版本用于回滚测试
- 提供版本变更日志

## 资源管理规范

### 文件命名规范
- **图标文件**：使用描述性名称，如 `multi_size.ico`
- **图片文件**：按功能或步骤编号命名
- **输出文件**：包含版本号和日期信息

### 文件大小优化
- **图标**：使用专业工具优化图标文件大小
- **图片**：压缩 PNG 图片，平衡质量和大小
- **可执行文件**：使用 UPX 等工具进行压缩

### 版本控制
- 资源文件变更需要更新版本号
- 重大变更需要备份旧版本
- 维护资源文件的变更日志

## 开发工作流

### 图标设计流程
1. **需求分析**：确定图标风格和尺寸需求
2. **设计制作**：使用专业工具创建图标
3. **格式转换**：生成多尺寸 ICO 文件
4. **测试验证**：在不同环境中测试显示效果
5. **优化发布**：优化文件大小和质量

### 图片制作流程
1. **内容规划**：确定每个步骤需要展示的内容
2. **界面截图**：从实际界面获取参考
3. **图形设计**：使用设计工具制作示意图
4. **格式优化**：保存为合适的图片格式
5. **集成测试**：在安装器中测试显示效果

### 发布准备流程
1. **资源检查**：确认所有资源文件就绪
2. **打包测试**：生成可执行文件并测试
3. **兼容性测试**：在不同系统上测试运行
4. **文档更新**：更新相关文档和说明
5. **版本发布**：准备发布包和更新说明

## 最佳实践

### 图标设计建议
1. **简洁性**：避免过于复杂的图案
2. **可扩展性**：设计时考虑不同尺寸的显示效果
3. **品牌一致性**：与扩展主题和品牌风格保持一致
4. **平台适配**：考虑 Windows 系统的图标显示规范

### 图片制作建议
1. **清晰度**：确保文字和图标清晰可读
2. **一致性**：保持所有图片的风格统一
3. **实用性**：图片内容应有助于用户理解安装步骤
4. **文件优化**：在保证质量的前提下优化文件大小

### 资源管理建议

2. **备份策略**：定期备份重要的资源文件
3. **文档同步**：资源变更时同步更新相关文档

# 组件模块说明

## 目录功能概述

`components/` 目录包含了浏览器扩展的所有 Vue 组件，采用组件化设计，提高代码复用性和可维护性。

## 文件说明

### 📋 hostList.vue
**主机列表组件**

**功能描述：**
- 显示和管理所有 VNC 主机列表
- 支持主机搜索、筛选和排序
- 提供主机连接、编辑、删除等操作
- 响应式设计，适配不同屏幕尺寸

**组件特性：**
- 支持虚拟滚动，优化大量主机的显示性能
- 实时显示主机连接状态
- 支持主机分组显示
- 提供批量操作功能

**Props 参数：**
- `hosts: Host[]` - 主机列表数据
- `searchKeyword: string` - 搜索关键词
- `loading: boolean` - 加载状态

**事件：**
- `@connect` - 连接主机事件
- `@edit` - 编辑主机事件
- `@delete` - 删除主机事件

**使用示例：**
```vue
<HostList 
  :hosts="hostList" 
  :search-keyword="searchText"
  :loading="isLoading"
  @connect="handleConnect"
  @edit="handleEdit"
  @delete="handleDelete"
/>
```

### 🏠 main.vue
**主应用组件**

**功能描述：**
- 扩展的主界面组件，包含完整的应用逻辑
- 集成主机列表、搜索、设置等功能
- 提供应用状态管理和事件处理

**组件特性：**
- 完整的应用生命周期管理
- 全局状态管理（Vuex/Pinia）
- 路由导航和页面切换
- 错误边界处理

**主要功能模块：**
1. **头部导航** - 应用标题、搜索框、用户信息
2. **侧边栏** - 功能菜单、快捷操作
3. **内容区域** - 主机列表显示和操作
4. **底部状态栏** - 连接状态、系统信息

**状态管理：**
- 管理主机列表数据
- 处理用户认证状态
- 控制应用主题和设置

## 组件设计原则

1. **单一职责**：每个组件只负责一个特定的功能
2. **可复用性**：组件设计考虑在不同场景下的复用
3. **Props 驱动**：通过 Props 传递数据，保持组件纯净
4. **事件通信**：使用自定义事件进行父子组件通信
5. **样式隔离**：使用 Scoped CSS 避免样式冲突

## 样式规范

- 使用 CSS Variables 实现主题切换
- 采用响应式设计，支持移动端适配
- 使用 Flexbox/Grid 布局系统
- 遵循 Material Design 设计规范

## 开发建议

1. 新组件应遵循现有的设计模式和命名规范
2. 复杂组件应拆分为多个子组件
3. 组件应提供完整的 TypeScript 类型定义
4. 重要组件应包含单元测试
